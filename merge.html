Merge Sort!

Merge sort uses what is known as a divide and conquer approach. This approach makes merge sorting more efficient than selection and insertion sorts.

Quick Facts

How it works:

Merge Sort calls itself (recursive) It splits itself into halves, than calls itself for each half, continuing to further split itself. After every division is sorted, they are all merged back together

Disadvantages.

Merge uses a temporary array that is as large as the array. This uses up more space

Big O notation

O(n * log(n)). Since merge sort is not affected by intial ordering, best and average case scenarios are often similar



Given that an array has more than 1 element, merge sort cuts the array into halves.
Then it sorts each half, hence "divide and conquer"
more like divide and sort. After the halves are sorted, we merge them
for a completely sorted array
