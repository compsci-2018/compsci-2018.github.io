Selection Sort: the "Search-and-swap" algorithm

Below is a table of must-known facts, and beyond that is a step-by step explanation of this algorithm. Enjoy!

Quick Facts

How it works:

find the smallest element, and swap it with a[0]... and so on

For an array of n...

the array is sorted after n-1 passes

After the kth pass..

the first k elements are in their final sorted position



tip: When we are dealing with a large n, selection sorts are not efficient

The selection sort SELECTS the SMALLEST element, and exchanges it with
the first element that hasnt aready been exchanged

Lets start with these elements:

9 7 2 4

The smallest element is 2! So now we exchange this with a[0]

2 7 9 4

Again, we look for the smallest element.
which is 4

Since we already exchanged an element at a[0], we move on to the next element at a[1].
Exchange 4 with 7

2 4 9 7

Getting the hang of it? 7 is our smallest element now. We already swapped with the element at a[0] and a[1]
so now lets exchange 9 and 7

2479
We're done! All elements are in order.
