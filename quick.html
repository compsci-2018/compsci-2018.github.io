Quick Sort!

If an array has more than 1 element
you partition the array(split the array into two subarrays)
from the "pivot" element which is usually the first element
The pivot is used to compare with other elements
All elements to the left of this pivot point
are less than the pivot element, and all elements to right are larger

Quick Facts

Fast run time

Splitting the array into two nearly equal sizes will result in the fastest run time.

Slow run time

On the other hand, if the pivot point is the smallest or largest, quicksort will be a slow and inefficient

Worst Case.

Occurs when quick sort keeps dividing an array of size 1 or n-1. To avoid this, shuffling the array can help
because this type of scenario occurs when an array is initially sorted and the 1st or last element is the pivot point
